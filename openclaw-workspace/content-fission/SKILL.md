# 内容裂变引擎 (content-fission)

## 触发词
`裂变` `内容裂变` `一鱼多吃`

## 核心理念

**一篇文章 = 一整周的全平台内容弹药**。

上班日只有2.5小时副业时间，裂变引擎从深度文章中
自动拆分出适合各平台的独立内容，一次创作七路分发。

## 架构

```
源文章(MD) → AI分析提取 → 裂变方案 → AI逐条生成 → 多平台发布 → 归档追踪
                ↑                                            ↓
            fission-engine.cjs                      content-analytics
            (调度+结构化)                            (数据回收飞轮)
```

**关键设计**：脚本负责调度和结构化，AI负责语义理解和内容生成。

## 前置条件

- 源文章：已完成的公众号深度文章（Markdown，800+字）
- 各平台 skill：toutiao-auto、zhihu-auto、wechat-md-publisher
- web-access CDP 可用
- 飞书通知已配置

## 完整流程

### Phase 0: 源文章输入

**输入方式**（三选一）：
1. 用户指定文件路径：`裂变 D:\path\to\article.md`
2. 用户贴内容或引用最近发布的文章
3. 从 Obsidian 发布归档中选最新的：`裂变 最新`

**验证**：
- 字数 ≥ 800
- 有明确的主题和论点结构

### Phase 1: AI 裂变分析

**由 AI 直接分析源文章**（不经过脚本），输出：

```
裂变方案：
├── 微头条 × 3
│   ├── #1: 金句型（最有共鸣的一句话 + 悬念引流）
│   ├── #2: 数据/案例型（最震撼的数据或案例）
│   └── #3: 互动型（抛出争议性观点引发讨论）
│
├── 知乎回答 × 1
│   └── 选最硬核的分论点，扩展为800-1500字知乎体
│
├── 小红书图文 × 1
│   └── 选最实用的方法/工具，300字小红书体
│
├── 星球精华帖 × 1
│   └── 只保留公众号没写的实操细节
│
└── 打卡金句 × 1
    └── 30字以内，有画面感的句子
```

**展示给用户确认**（除非用户说 `裂变 全部`）

### Phase 2: AI 逐条生成

AI 为每个裂变产物单独生成内容，遵循各平台规则：

#### 微头条规则
- 100-200字，口语化短句
- 结尾自然引到公众号「赛博番茄」
- 带1-2个话题标签
- 禁止标题党、AI味

#### 知乎回答规则
- 800-1500字，先结论后论证
- 找到匹配的知乎问题（搜索或用户提供）
- 不带公众号引流（知乎禁忌）

#### 小红书规则
- 标题≤20字，带emoji
- 正文≤300字，要点式
- 自动生成封面图
- 3-5个热门话题

#### 星球精华帖规则
- 500-800字
- 定位「公开文章没写的实操细节」
- 结尾抛互动问题

#### 打卡金句规则
- ≤30字，有观点有画面感
- 用于头条/知乎每日打卡

### Phase 3: 多平台发布

**自动发布**：微头条（间隔5分钟）+ 打卡金句
**需确认**：知乎回答、小红书、星球精华帖
**紧急模式**：`裂变 全部` 跳过所有确认

使用各平台 skill：
- toutiao-auto → 微头条发布
- zhihu-auto → 知乎回答发布
- wechat-md-publisher → （源文章本身）
- xhs-scraper + CDP → 小红书发布（如有）

### Phase 4: 归档

归档到 Obsidian：`04-发布归档/裂变记录/{日期}-{文章简称}.md`

格式：
```markdown
# 裂变记录：{源文章标题}
> 日期：{YYYY-MM-DD}

## 源文章
- 标题：{title}
- 字数：{count}
- 原文链接：{公众号链接}

## 裂变产物
| 平台 | 类型 | 字数 | 状态 |
|------|------|------|------|
| 头条 | 观点型微头条 | 150 | ✅ 已发布 |
| 头条 | 数据型微头条 | 180 | ✅ 已发布 |
| 知乎 | 回答 | 1200 | ✅ 已发布 |
| 小红书 | 图文 | 280 | ⏳ 待确认 |
| 星球 | 精华帖 | 650 | ⏳ 待确认 |
| 头条 | 打卡金句 | 30 | ✅ 已发布 |

## 数据追踪（7天后更新）
- 微头条阅读量：-
- 知乎赞同数：-
```

### Phase 5: 数据飞轮（7天后）

追踪各裂变产物的数据表现，写入 content-analytics，
反哺内容工厂的选题评分。

## 命令速查

| 命令 | 动作 |
|------|------|
| `裂变` | 选择源文章，交互式裂变 |
| `裂变 {路径}` | 指定源文章开始裂变 |
| `裂变 最新` | 从发布归档选最新文章 |
| `裂变 全部` | 跳过确认，全平台自动发布 |
| `裂变 微头条` | 只裂变头条内容 |
| `裂变 知乎` | 只裂变知乎回答 |
| `裂变 追踪` | 查看最近裂变数据 |

## 关键规则

1. **质量优先**：每个裂变产物独立有价值，不是简单截取
2. **平台适配**：调性/字数/格式各平台匹配
3. **公众号护城河**：最精华内容留在公众号，裂变产物是钩子
4. **节奏控制**：微头条间隔5分钟发布
5. **AI驱动**：脚本只调度，内容质量由AI保证
