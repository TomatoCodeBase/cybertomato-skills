# 内容工厂引擎 (content-factory-engine)

## 触发词
`内容工厂` `开工厂` `自动写稿` `内容流水线`

## 前置条件
- Obsidian运行中（localhost:27123）
- 微信公众号凭证已配置（.env: WECHAT_APP_ID, WECHAT_APP_SECRET）
- 封面图API已配置（.env: DOUBAO_API_KEY 或 DASHSCOPE_API_KEY）

## 三种模式

### 模式1：手动触发（默认）
```
用户: 内容工厂
→ 扫描灵感库 → AI推荐选题 → 用户选 → 自动成文 → 推草稿
```

### 模式2：定时批量
```
cron触发（每日9:00/21:00）
→ 自动从灵感库/热榜捞素材 → 自动评分选题 → 批量成文 → 推草稿 → 飞书通知
```

### 模式3：全自动
```
cron触发（每4小时）
→ 自动采集热榜 → 灵感评分 → 选题决策（≥80分自动写） → 成文 → 质量门控 → 推草稿 → 飞书通知
```

## 流程（模式1：手动触发）

### Step 1: 扫描素材库
扫描两个来源，列出可用素材：

**1a. 灵感库未处理素材**
```bash
node scripts/obsidian-api.cjs list "01-灵感素材库/1-日常灵感剪报/"
node scripts/obsidian-api.cjs list "02-选题仓库/待写选题库/"
```

**1b. 热榜最新素材（可选）**
- 小红书热搜：使用 `xhs-scraper` skill
- 知乎热榜：使用 `zhihu-hot.cjs`

**输出格式**：
```
📊 素材盘点
━━━━━━━━━━━━━━━━
📁 灵感库: X篇未处理
  1. [25/30] AI Agent实战指南（20260403）
  2. [22/30] 微信排版技巧（20260402）
  ...
📁 选题库: X篇待写
  1. [待定] 我用AI搭了一条内容流水线
  ...
📁 热榜: X条新话题（关键词：AI、副业、公众号）
━━━━━━━━━━━━━━━━
```

**Success criteria**: 列出所有可用素材，带评分和日期

### Step 2: AI推荐选题
对素材进行综合评分，推荐Top 3选题。

**评分算法**（复用 topic-scorer skill 的5维度）：
- 时效性 20分
- 实用性 20分
- 独特性 20分
- 受众面 20分
- 情感共鸣 20分

**额外加分**（方案C特有）：
- 与历史爆款主题相似 +5分
- 契合番茄文风 +5分
- 有现成素材支撑 +5分

**输出**：
```
🤖 AI推荐选题（Top 3）
━━━━━━━━━━━━━━━━
🥇 [85/100] 我用AI搭了一条内容流水线，效率提升300%
   → 素材来源：灵感库(20260403) + 热榜AI话题
   → 推荐角度：第一人称实操+数据对比
🥈 [78/100] 公众号排版3个致命错误，90%的人都犯过
   → 素材来源：选题库
   → 推荐角度：避坑指南+截图对比
🥉 [72/100] AI时代，普通人的副业机会在哪里
   → 素材来源：热榜+知乎热榜
   → 推荐角度：趋势分析+实操路径
━━━━━━━━━━━━━━━━
💬 选一个，或说"全写"
```

**Human checkpoint**: 等待用户选择

**Success criteria**: 推荐Top 3选题，每个带评分、素材来源、推荐角度

### Step 3: 拉取素材
用户选题后，自动拉取所有相关素材。

```bash
# 从灵感库/选题库读取素材
node scripts/obsidian-api.cjs get "01-灵感素材库/1-日常灵感剪报/YYYYMMDD_素材标题.md"
# 从素材片段库读取参考
node scripts/obsidian-api.cjs list "01-灵感素材库/2-爆款素材片段/"
```

**强制规则**：成文前必须拉素材，禁止裸写。

**Success criteria**: 所有相关素材内容已读取，展示给用户确认

### Step 4: 成文（含中间检查点）
基于素材 + 番茄文风 + 维的万法成文。

**写作规范**：
1. 读取 `tomato-writing` skill 获取风格指南
2. 读取维的万法：`00-个人日记/提示词库/维的万法.md`
3. 结构：标题→痛点→核心观点（数据+案例）→方法步骤→工具技巧→行动建议
4. 字数：1500-3000字
5. 禁止词：总而言之、不可否认、综上所述、不难发现、众所周知
6. 推荐词：说白了、实际上、换个角度、关键是、核心在于

**中间检查点（新增，嵌入写作过程中）**：

| 检查点 | 时机 | 检查内容 | 不通过处理 |
|--------|------|---------|-----------|
| CP1 选题对齐 | 确定选题后、写作前 | 选题是否锚定用户意图？与素材匹配度如何？ | 调整角度或重新选题 |
| CP2 大纲校验 | 大纲生成后、正文前 | 结构是否完整（痛点→方法→建议）？论断是否有支点？ | 补充缺失模块 |
| CP3 中间校验 | 正文写到60%时 | 是否跑偏原定大纲？信息密度是否达标（每段有新信息）？ | 回锚修正 |
| CP4 终检 | 全文完成后 | 见下方质量自检清单 | 逐项修复直至通过 |

**质量自检（CP4 终检，成文后自动执行）**：
- [ ] 字数在1500-3000之间
- [ ] 无禁止词
- [ ] 包含至少2个生活化比喻
- [ ] 开头3句话入题
- [ ] 结尾有行动建议
- [ ] 标题包含数字或疑问
- [ ] 一级标题 ≠ 文章标题/文件名
- [ ] 每个关键论断有2-3个支撑（数据/类比/案例）
- [ ] 商业短文：白描开局 + 教程编号 + 预判疑问（适用时）

**Success criteria**: 成文通过CP1-CP4全部检查，保存到初稿打磨区

### Step 5: 预览确认
将初稿转为微信HTML，生成预览。

```bash
python skills/wechat-md-publisher/scripts/create_preview.py --input "temp/draft.md"
```

推送预览给用户（飞书卡片/直接发送），等待确认。

**Human checkpoint**: 用户确认或要求修改

**Success criteria**: 用户明确说"发布"或"推草稿"

### Step 6: 推草稿+归档
用户确认后自动执行。

1. **生成封面**：
```bash
python skills/wechat-article-publisher/scripts/generate_cover.py --prompt "封面描述"
```

2. **转HTML + 推草稿**：
```bash
python skills/content-factory-publisher/scripts/content_factory_publish.py --draft "文章标题.md"
```

3. **自动归档**：
```bash
node scripts/archive-published-article.cjs "文章标题"
```

**Success criteria**: 草稿ID已返回，文章已归档，YAML状态已更新

## 流程（模式2：定时批量）

### 触发条件
cron任务，每日9:00和21:00执行。

### 自动流程
```
1. 扫描灵感库 → 列出评分≥20的素材
2. 按评分排序 → 取Top 3
3. 批量成文（无需用户确认选题）
4. 批量推草稿
5. 飞书通知用户：
   "🏭 内容工厂日报
    ✅ 已生成 X 篇文章
    📝 草稿1: 标题 | 评分XX
    📝 草稿2: 标题 | 评分XX
    📝 草稿3: 标题 | 评分XX
    👉 请在公众号后台审核发布"
```

### 批量模式规则
- 不需要用户确认选题（信任评分算法）
- 不需要用户确认成文（信任质量门控）
- 用户只需审核草稿后发布
- 低于70分的选题不写，移入回收站

## 流程（模式3：全自动）

### 触发条件
cron任务，每4小时执行一次（6:00, 10:00, 14:00, 18:00, 22:00）。

### 自动流程
```
1. 采集热榜素材（小红书+知乎）
   → 发现新话题自动入库
2. 灵感评分
   → ≥80分 → 自动进入选题库
   → 60-79分 → 通知用户"发现选题机会"
   → <60分 → 丢弃
3. 选题决策
   → 有≥80分选题 → 自动成文
   → 无 → 跳过，等下一轮
4. 成文 + 质量门控
   → 通过 → 推草稿
   → 未通过 → 重写1次 → 仍不通过 → 丢弃并记录教训
5. 飞书通知
```

### 全自动模式质量门控
自动模式下，成文后必须通过以下检查：
- [ ] 字数≥1200
- [ ] AI味检测（禁止词列表+AI常用句式检测）
- [ ] 标题吸引力评分≥70
- [ ] 无事实性错误（需交叉验证关键数据）

## 质量门控（Quality Gate）

### AI味检测规则
```
禁止句式：
- "总而言之"、"综上所述"、"不可否认"
- "在当今社会"、"随着XX的发展"
- "值得注意的是"、"不难发现"
- "众所周知"、"不言而喻"
- "首先...其次...最后..."（超过2层）
- 每段开头用"然而"、"此外"超过3次

AI特征词频率限制：
- "赋能" ≤ 1次
- "生态" ≤ 2次
- "矩阵" ≤ 1次
- "闭环" ≤ 1次
- "底层逻辑" ≤ 1次
```

### 标题吸引力评分
```
- 包含数字：+10
- 包含疑问/悬念：+10
- 包含"我"（第一人称）：+5
- 包含具体结果/数据：+15
- 字数10-25字：+10
- 包含对比/冲突：+10
- 总分≥70：通过
```

## 偏好学习模型（方案C核心）

### 历史数据分析
每次发布后自动学习：
1. 从 `04-发布归档/公众号已发布/` 读取历史文章
2. 提取特征：标题模式、字数分布、话题分布、结构模式
3. 更新偏好文件：`memory/content-factory-preferences.json`

### 偏好特征
```json
{
  "topicPreference": ["AI工具", "副业实操", "技术教程"],
  "titlePattern": ["我用...+数据结果", "X个致命错误", "AI时代+机会"],
  "wordCountRange": [1500, 2500],
  "structurePreference": "痛点→方法→工具",
  "toneWords": ["说白了", "关键是", "核心在于"],
  "forbiddenPatterns": ["总而言之", "综上所述"],
  "historicalScores": [],
  "bestPerformingTopics": []
}
```

### 偏好应用
选题评分时，与偏好特征匹配度高的自动加分。

## 已知陷阱（2026-04-05实战记录）
1. **YAML缺失**：从选题库成文时，必须先读取YAML元数据再写入初稿区
2. **封面API限流**：豆包API每分钟限制5次，批量模式下需加延迟
3. **Obsidian API超时**：大量文件列表时可能超时，分页查询
4. **微信草稿内容大小**：图文消息content字段限制20KB，长文需分段或压缩HTML
5. **编码问题**：所有Obsidian文件必须UTF-8无BOM，参考ERR-20260403-001
