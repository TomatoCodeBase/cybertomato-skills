# 内容工厂发布助手

## 任务目标

从 Obsidian 内容工厂读取初稿，转换为微信 HTML，推送到草稿箱，自动归档。

**核心功能**：
- ✅ 读取 Obsidian 初稿（`03-内容工厂/在写/初稿`）
- ✅ 转换为微信 HTML（v3，代码块完美显示）
- ✅ 推送到草稿箱（支持预览/发布模式）
- ✅ 自动归档（移动到 `04-发布归档/公众号已发布`）
- ✅ 更新内容工厂状态
- ✅ **"已发布"触发词：用户确认发布后自动归档**

**触发词**：`内容工厂发布`

**契约书**：`内容工厂发布`

## 操作步骤

### 流程 1：预览模式（推荐先预览）

**使用场景**：查看初稿在微信中的效果，不推送到草稿箱

**命令**：
```bash
python scripts/content_factory_publish.py --preview "03-内容工厂/在写/初稿/文章标题.md"
```

**智能体操作**：
1. 用户说："内容工厂预览 文章标题"
2. 从 Obsidian 读取初稿
3. 转换为微信 HTML
4. 用浏览器打开预览
5. 告知用户确认效果

**使用场景**：用户在公众号后台确认发布后，自动归档到已发布区并生成创作日志

**触发词**：`已发布`

**执行动作**：
1. 查找初稿打磨区的文件

6. **自动同步到飞书多维表格**（如果配置了）

```bash

```

**Obsidian 仓库路径**：`D:\cybertomato`（2026-03-23 迁移更新）

**读取逻辑**：
```python
obsidian_base = r"D:\cybertomato"
full_path = os.path.join(obsidian_base, draft_path)

with open(full_path, 'r', encoding='utf-8') as f:
    content = f.read()
```

**转换器**：`skills/wechat-md-publisher/scripts/markdown_to_wechat_v2.py`（v3）

## 注意事项

### 文件命名

**初稿命名规范**：
- ✅ 使用中文或英文
- ✅ 不要包含特殊字符（`/ \ : * ? " < > |`）
- ✅ 推荐格式：`文章标题.md`

### 内容工厂流程

**完整流程**（SOP_GZH.md）：
```
灵感捕获 → 选题生成 → 初稿打磨 → 审稿发布 → 已发布归档
```

**本 Skill 负责的部分**：
```
初稿打磨 → 审稿发布 → 已发布归档
```

### 归档后操作

**归档后需要手动操作**：
1. 在公众号后台确认草稿
2. 发布文章
3. 检查文章链接
4. 更新内容工厂状态（可选）
