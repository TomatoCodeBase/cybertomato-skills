# 微头条生成技能

> 前置依赖：`ai-daily-report-merge` 产出的融合日报。

## 核心原则

**内容优先级（铁律）：**
- 普适性洞见 > 情境性观点
- 思维模式 > 具体结论
- 可迁移智慧 > 专属经验

不是"告诉读者发生了什么"，而是"让读者带走一个能复用的认知"。

## 选材规则

- 从融合日报 🔥今日重点 + 🛠️产品与技术 中选 **2-3条**
- **硬性上限3条，不可超过**
- 优先选：对普通人的工作/赚钱方式有实际影响的
- 避免选：纯行业八卦、估值数字、人事变动
- 宁缺毋滥——只有1条值得写就只写1条

## 单条结构（300-500字）

```
## 标题（口语化，有观点，不带分级标签）

第一段：发生了什么（2-3句事实，不啰嗦）

第二段：说人话解读（"这意味着什么"/"说人话就是..."）
→ 这里放可迁移的认知，不是这件事的结论，而是这类事的规律

第三段：实际启发（"如果你是XX，现在就该YY"）
→ 给出具体可执行的动作建议，哪怕很小
```

## 风格要求

- **口语化**：像跟朋友聊天，不用"据悉""值得关注"
- **有态度**：敢下判断，不做新闻复读机
- **说人话**：技术概念用一句话解释清楚
- **短句为主**：每句不超过20字，超过就拆
- **禁止**：分级标签（S级/A级）、"值得关注"、"据悉"、"引发关注"

## 标题写法

- ✅ `Cloudflare用AI裁掉1100人，收入反创新高`
- ✅ `DeepSeek豪掷200亿启动500亿融资，V4.1六月发布`
- ✅ `ChatGPT 5.5 Pro一小时搞定博士级数学题`
- ❌ `🎯 S级：支付宝AI付实现"AI代买"`
- ❌ `AI行业重大突破！谷歌发布新模型`

```markdown
正文...

正文...

```

## 10W+ 爆文规律（从头条科技领域10W+学习提炼）

### 选材规律
- **找"反直觉"的切入点**：不是"AI很厉害"，而是"AI的瓶颈不是芯片是变压器"；不是"芯片很重要"，而是"2nm不是唯一出路"
- **具体数字 > 抽象描述**："20万一台的变压器" > "昂贵的电力设备"；"3亿月活" > "用户量很大"
- **让读者产生"原来如此"的感觉**：用一个事实推翻一个常识

### 标题规律
- **制造悬念或反转**："未来全球争抢的并非芯片，是中国20万一台的变压器"
- **用具体场景替代抽象概念**："亲驾FSD后背心发凉" > "自动驾驶体验很好"
- **暗示读者不知道某个重要信息**："XXX终于撕下伪装""你还在XXX，其实已经XXX"
- **标题即钩子**：读者看到标题就必须点进来看答案

### 结构规律
- **开头就是冲击**：第一句就给出反常识/悬念/数据，不要铺垫
- **用"说人话"桥接**：技术内容后跟一句"说人话就是..."，降低门槛又不降智
- **层层递进**：现象→原因→本质→启发，每一层比上一层更深
- **结尾必有"带走的东西"**：不是一个结论，而是读者能用在自己身上的认知

### 写作节奏
- **长句拆短**：每句不超过20字，超过就拆成两句
- **段间留白**：一个想法一段，不要堆砌
- **适度情绪词**："后背发凉""荒诞""神来之笔"——有情绪才有传播，但不能太多
- **制造"我和你说个秘密"的对话感**：不是在写文章，是在跟朋友说一件他不知道的事

### 可迁移认知（为什么这些内容能爆）
1. **信息差**：读者看了觉得自己"知道了别人不知道的事"
2. **认知升级**：不是多了一条新闻，而是换了一个看问题的角度
3. **身份认同**：让读者觉得"我关注科技/AI是有前瞻性的"
4. **可传播性**：读者转发时附上一句"你看这个"，内容本身帮他表达了观点

## 流程

1. 读取当日融合日报
2. 按选材规则选出2-3条
3. 每条按结构改写（300-500字）
4. 校验：是否带走了可迁移的认知？是否给出了可执行的建议？
5. 写入存档路径
