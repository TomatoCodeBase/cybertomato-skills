# web-site-ops — 网站操作经验库

**用途**：积累各网站的操作路径、页面结构、登录方式和踩坑记录，减少每次浏览器自动化的试错成本。

**触发条件**：需要对某个网站执行浏览器操作时，先查本 skill 对应的站点经验。

---

## 使用规则

1. **操作前先读**：去 `sites/<domain>.md` 找已有经验
2. **操作后沉淀**：每次浏览器操作完成后，把新发现追加到对应文件
3. **格式统一**：每个站点文件遵循模板
4. **不过度记录**：只记对自动化有价值的信息（按钮位置、操作路径、登录方式、防爬策略、页面加载特点），不记内容本身

---

## 站点经验模板

```markdown
# <站点名> — <domain>

## 基本信息
- 用途：一句话描述
- 登录方式：微信扫码 / 手机验证码 / 账号密码 / OAuth
- 是否需要登录才能操作：是/否

## 页面结构要点
- 关键页面 URL 和路径
- 页面加载特点（SPA / SSR / JS 渲染 / 懒加载）
- 关键按钮/元素的稳定定位方式

## 操作路径
### <操作名>
1. 步骤 1
2. 步骤 2

## 踩坑记录
- <日期>：<问题描述 + 解决方案>

## 更新日志
- <日期>：初始记录
```
