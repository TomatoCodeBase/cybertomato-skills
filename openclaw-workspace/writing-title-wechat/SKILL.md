# 公众号爆款标题生成器

基于心理学原理和传播学规律的标题生成技能。

---

## 输出格式

| 序号 | 类型 | 标题内容 | 情绪词 | 预估点击率 | 设计逻辑 |
|:---|:---|:---|:---|:---|:---|

**数量**：5个单模型 + 5个混合模型 = 10个标题

**点击率范围**：7.0-9.5（微信生态标准）

## 12大爆款模型

| 模型 | 核心原理 | 常用情绪词 |
|------|----------|------------|
| 1. 痛点焦虑型 | 损失厌恶，恐惧驱动 | 危机、警报、可怕、崩溃 |
| 2. 利益承诺型 | 具体收益，趋利本能 | 赚翻、省钱、秘诀、攻略 |
| 3. 反常识型 | 认知失调，打破预期 | 居然、竟然、反转、颠覆 |
| 4. 故事悬念型 | 信息缺口，好奇驱动 | 惊呆、泪目、震惊、揭秘 |
| 5. 人群标签型 | 身份认同，代入感 | 写给、致、所有、必读 |
| 6. 数字量化型 | 锚定效应，认知边界 | 3个、5招、90%、10倍 |
| 7. 对比反差型 | 对比效应，视觉冲击 | vs、却、差距、截然不同 |
| 8. 权威背书型 | 权威效应，可信度 | 专家、研究、哈佛、清华 |
| 9. 时效紧迫型 | 稀缺性，行动驱动 | 最后、限时、仅剩、抓紧 |
| 10. 情感共鸣型 | 情感触发，深层共鸣 | 泪目、心酸、温暖、扎心 |
| 11. 亲身经历悬念型 | 真实经历+悬念+数字结果 | 才知道、居然、翻了、暴涨 |
| 12. 认知决策困境型 | 认知盲区+二元对立 | 该不该、要不要、精华、有害 |

## 核心原则

**点击意愿公式**：
```
点击意愿 = （预期价值 × 需求迫切度）÷ 认知成本
```

**关键发现**：
- 明确负面 > 模糊正面（损失敏感度是收益的2.5倍）
- 专属感 > 大众化（精准锁定目标用户）
- 数字 = 认知边界（降低认知负荷）
- 中等偏上情绪唤醒（过高触发防御，过低无法激活）

**详细说明**：见 [references/click-behavior.md](references/click-behavior.md)

用户直接触发「爆款标题」时：

1. **生成10个标题**（按标准格式：5单模型+5混合模型）
2. **自动保存到默认路径**（不二次确认）：
   - 文件夹：`D:\cybertomato\03-内容工厂\参考\爆款标题\{主题关键词}\`
   - 文件：`候选标题.md`
3. **告知用户保存路径**

## 完整流程（三步筛选法）

无论是独立调用还是被其他技能联动，都遵循同一流程：

### 第一步：生成10个候选标题

1. **读取文章内容**（或用户提供原文/主题）
2. **生成10个标题**（5单模型+5混合模型）
3. **输出表格格式**：
   ```
   | 序号 | 类型 | 标题内容 | 情绪词 | 预估点击率 | 设计逻辑 |
   ```
4. **用户选择一个** → 进入第二步

### 第二步：3个最终方案

用户选中一个标题后，基于该标题微调生成3个最终方案：

- **#1**：原版（用户选中的标题）
- **#2**：微调A（调整角度/情绪/用词）
- **#3**：微调B（另一种微调方向）

直接输出3个方案，不解释为什么这么改。

用户选择最终标题 → 进入第三步

### 第三步：定标题

1. 在候选标题文件中追录完整流程
2. 保存到默认路径（严格2层，禁止建子文件夹）：
   - 保存到：`D:\cybertomato\03-内容工厂\参考\爆款标题\候选标题_{主题关键词}.md`
3. **文件模板**（严格按此格式）：
   ```markdown
   # 候选标题：{文章主题}

   > 生成日期：{YYYY-MM-DD}
   > 原文：{关联的原文路径}

   ## 10个候选标题

   | 序号 | 类型 | 标题内容 | 情绪词 | 预估点击率 | 设计逻辑 |
   |:---|:---|:---|:---|:---:|:---|
   | 1 | xxx | xxx | xxx | 8.5 | xxx |
   | ... |
   | 10 | xxx | xxx | xxx | 9.0 | xxx |

   ## 🏆 人工筛选优化（3个最优方案）

   > 原始选择：{用户选中的原始标题}

   | 序号  | 最终标题                                      |
   | :-: | :---------------------------------------- |
   |  1  | xxx |
   |  2  | xxx |
   |  3  | xxx |

   ## ✅ 最终选用
   **#N** 最终标题内容
   ```
4. 告知用户保存路径

**完整流程：10初稿 → 用户选择 → 3优化 → 用户选择 → ✅最终选用**

## 质量检查清单

- [ ] 标题长度：14-30个汉字（口语化标题可以稍长）
- [ ] 口语化：读出来像朋友聊天，不像技术文档
- [ ] 结构完整：前半句给信息，后半句给行动/收益
- [ ] 内容忠实：忠实于原文主旨，禁止"标题党"
- [ ] 关键词自然嵌入：品牌/核心词出现但不僵硬
- [ ] 可信感 > 刺激感：不用"震惊""可怕"等过载情绪词

## 从实战中提炼的选择偏好（4篇定稿分析）

> 基于4次真实选择抽象出的规律，比理论模型更可靠

### 1. 口语化 > 书面语
```
❌ "底层架构重构与任务流回归" → 像技术文档
✅ "这次更新有点东西" → 像朋友聊天
```
**模式**：用日常口语表达，降低认知门槛

### 2. 陈述事实 + 引导行动 > 纯情绪刺激
```
❌ "震惊！你的AI助手可能早就挂了" → 情绪过载，像标题党
✅ "更新有点东西，点个关注，下个版本我第一时间告诉你" → 有信息+有行动
```
**模式**：前半句给信息，后半句给读者一个动作

### 3. 承诺通用价值 > 自限具体形式
```
❌ "一张表看完" → 限死了内容厚度
✅ "一文看完所有变化" → 开放式承诺
```
**模式**：用"一文/搞定/看完"代替"一张表/3步/X招"

### 4. 关键词必现，但自然嵌入
```
❌ "OpenClaw 2026.4.1更新有点东西" → 品牌词在句首，僵硬
✅ "OpenClaw 2026.4.2：不升模型改底层，这篇文章帮你一文看完所有变化" → 冒号后展开
```
**模式**：品牌/核心词出现，但融入句式结构，不像广告

### 5. "有点东西"是高频选中信号
3篇中1篇直接用了"有点东西"，这是番茄的口癖
**模式**：在微调阶段优先保留原版中的口语化表达

### ⚠️ 关键纠偏：预估点击率 ≠ 真实选择
- 4篇最终选中的标题都不是预估点击率最高的
- 质量检查清单中的"情绪词必须强烈"这条已失效
- 真实偏好：**可信感 > 刺激感**

### 6. 精炼压缩 > 冗余堆砌（第4篇定稿）
```
❌ "Hermes5步极简懒人安装，从申请API到迁移OpenClaw旧数据一文教你全搞定" → "极简"和"懒人"重复，"教你"多余
✅ "Hermes懒人5步安装，从API申请到OpenClaw数据迁移一文搞定" → 删冗余、调语序、更紧凑
```
**模式**：微调阶段优先做减法——去掉重复修饰词（极简+懒人→懒人）、调整语序更顺口（申请API→API申请）、砍掉不必要的动词（教你全搞定→一文搞定）。好的标题是削出来的，不是堆出来的。

## 搜索词补全

品牌+型号后，补上品类词，覆盖更多搜索长尾。
```
❌ OpenClaw 3个GLM-5怎么选    → 搜"GLM-5模型"的人搜不到
✅ OpenClaw 3个GLM-5模型怎么选 → 覆盖"GLM-5"和"GLM-5模型"两个关键词
```

## 禁止

- ❌ 夸大、欺骗、标题党
- ❌ 标题承诺 ≠ 内容兑现
- ❌ 滥用被滥用的元素（如"震惊！"）
- ❌ 自限价值承诺（用"一张表""3个步骤"限定内容厚度）
- ❌ 生硬书面语（"底层架构重构"不如"底层改了什么"）
- ❌ 情绪过载（"可怕""崩溃""震惊"等词已被反复淘汰）
