# 微头条生成技能

> 前置依赖：`ai-daily-report-merge` 产出的融合日报。

## 核心原则

**内容优先级（铁律）：**
- 普适性洞见 > 情境性观点
- 思维模式 > 具体结论
- 可迁移智慧 > 专属经验

不是"告诉读者发生了什么"，而是"让读者带走一个能复用的认知"。

## 选材规则

- 从融合日报 🔥今日重点 + 🛠️产品与技术 中选 **2-3条**
- **硬性上限3条，不可超过**
- 优先选：对普通人的工作/赚钱方式有实际影响的
- 避免选：纯行业八卦、估值数字、人事变动
- 宁缺毋滥——只有1条值得写就只写1条

## 单条结构（300-500字）

```
## 标题（口语化，有观点，不带分级标签）

第一段：发生了什么（2-3句事实，不啰嗦）

第二段：说人话解读（"这意味着什么"/"说人话就是..."）
→ 这里放可迁移的认知，不是这件事的结论，而是这类事的规律

第三段：实际启发（"如果你是XX，现在就该YY"）
→ 给出具体可执行的动作建议，哪怕很小
```

## 风格要求

- **口语化**：像跟朋友聊天，不用"据悉""值得关注"
- **有态度**：敢下判断，不做新闻复读机
- **说人话**：技术概念用一句话解释清楚
- **短句为主**：每句不超过20字，超过就拆
- **禁止**：分级标签（S级/A级）、"值得关注"、"据悉"、"引发关注"

## 标题写法

- ✅ `Cloudflare用AI裁掉1100人，收入反创新高`
- ✅ `DeepSeek豪掷200亿启动500亿融资，V4.1六月发布`
- ✅ `ChatGPT 5.5 Pro一小时搞定博士级数学题`
- ❌ `🎯 S级：支付宝AI付实现"AI代买"`
- ❌ `AI行业重大突破！谷歌发布新模型`

## 输出格式

```markdown
# YYYY-MM-DD 微头条存档

## 微头条1：标题

正文...

## 微头条2：标题

正文...

## 微头条3：标题（可选）

正文...
```

## 输出路径

`D:\cybertomato\03-内容工厂\日报\微头条存档\YYYY-MM-DD-微头条.md`

## 输入路径

融合日报：`D:\cybertomato\03-内容工厂\日报\Ai融合日报\YYYY-MM-DD-番茄AI热点日报.md`

## 流程

1. 读取当日融合日报
2. 按选材规则选出2-3条
3. 每条按结构改写（300-500字）
4. 校验：是否带走了可迁移的认知？是否给出了可执行的建议？
5. 写入存档路径
