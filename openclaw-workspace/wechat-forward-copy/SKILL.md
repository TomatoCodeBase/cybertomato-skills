# 微信转发文案生成

## 输入
1. 文章URL或正文内容
2. 关键词（必须出现在文案中，通常是产品/项目名）
3. 文案类型：朋友圈 / 群聊转发 / 两者都要
4. 数量（默认各5条）

## 文案心法（提炼自微信朋友圈与公众号文案.md）

### 底层逻辑
- 文案不是"写文字"，是"设计社交互动"——价值交换
- 前3秒决定生死，开头直接切入读者最关心的点
- 情绪优先于理性，但有用的文案比有趣的更长久

### 5种核心思维模式
1. **痛点放大**：把痛点具体化、场景化，再给解法
2. **结果前置**：对方最关心的结果放第一句
3. **对比反差**：没有对比就没有认知（旧方案 vs 新方案）
4. **故事化**：用"我"的经历，一句话讲完一个顿悟瞬间
5. **身份认同**：写读者想成为的样子

### 铁律
- **1-2句话，字数越少越好**，能一句说清绝不用两句
- **关键词必须出现**，不能遗漏
- 朋友圈偏个人叙事和情绪，群聊偏实用和社交货币
- 不说废话：砍掉"大家好""分享一篇""今天给大家推荐"
- 不解释：读者不需要懂原理，只需要感受到价值

### 文案结构模板（1-2句版）

**朋友圈句式：**
- 痛点+转折：`[具体痛点场景]。[关键词]：[一句话解法/结果]`
- 顿悟型：`用了[关键词]才发现，[颠覆认知的结论]`
- 对比型：`[旧方案]做不到[X]，[关键词]做得到`

**群聊转发句式：**
- 利他型：`[关键词][一句话价值]，0成本/X分钟搞定`
- 社交认同型：`试了[关键词]，[具体收益]，分享给同样[痛点]的群友`
- 极简型：`[关键词]——[一句话核心卖点]`

## 输出格式

```
朋友圈 × N

1. 文案1
2. 文案2
...

微信群转发 × N

6. 文案1
7. 文案2
...
```

## 流程

1. 提取文章核心卖点和关键信息（用 wechat-article-extract 或直接读取正文）
2. 识别目标受众的痛点和渴求
3. 确认用户给的关键词
4. 按上述5种思维模式各写至少1条，凑够数量
5. 用户挑选后，可选保存为技能的参考案例

## 用户选过的优质文案（案例库）

### 案例1：OpenClaw+Obsidian个人任务系统文章

**朋友圈精选：**
> OpenClaw+Obsidian个人任务系统：时间戳一打，统计的清清楚楚。

**群聊转发精选：**
> OpenClaw+Obsidian个人任务系统，这套让你知道时间去向。

分析：两条都把关键词前置，一句话完成"是什么+为什么值"，零废话。

### 案例2：Obsidian任务模板+个人效率系统搭建

**朋友圈终稿：**
> 个人效率系统搭建没你想的那么难，一套Obsidian任务模板就够了。关注番茄公众号，发送模板，免费领取。

分析：三段式结构——痛点破除+方案给出+行动指令（关注+关键词+免费），关键词覆盖"个人效率系统搭建"和"Obsidian任务模板"两个搜索词，CTA明确且低门槛。
