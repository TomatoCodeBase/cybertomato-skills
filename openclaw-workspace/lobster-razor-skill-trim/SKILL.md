# 龙虾剃刀 · SKILL.md精简

## 何时使用
- SKILL.md超过15KB（加载时占注意力权重）
- 文件含大量官方文档粘贴
- 同一规则重复出现3次以上
- 常识性描述占比过高

## 龙虾剃刀三道检验

每句话过三关，任一不通过就删：

1. **删掉行为会变吗？** 不会→删
2. **个人化还是常识？** 常识→删（模型已内化）
3. **重复意图？** 合并为最锋利一条

## 两种bloat类型与策略

### 类型A：文档dump（如pytorch-fsdp 160KB）

特征：直接粘贴官方API文档、教程、参数说明表
策略：**核弹级裁剪**，砍90%+

- 官方文档 → 删，改为链接行
- API签名/参数表 → 删，模型已训练过
- 通用教程示例 → 删，保留1个最精简的
- **保留**：决策表（选A还是B）、踩坑清单、版本兼容要求

模板：
```
## 核心API速查（代码块，不解释）
## 关键决策点（表格：场景→推荐）
## 踩坑（编号列表，只写反直觉的）
## 官方文档（链接，不内联）
```

### 类型B：实战积累（如zsxq-auto 52KB）

特征：多次实战迭代追加的踩坑经验、选择器、字段ID
策略：**合并去重**，砍40-60%

- 同一坑重复3次 → 合并为1条最锋利版
- 代码块前的解释文字 → 删，代码自解释
- "⚠️致命"反复出现 → 保留真的致命的，其余降级
- 已废弃方案（如XHR withCredentials失效） → 压缩为1行"❌ 方案X已失效，原因Y"
- 重复的流程描述（如Cron架构出现两次） → 合并
- **保留**：选择器、字段ID、API端点、URL模板、去重逻辑、降级策略

## 执行步骤

1. `wc -c` 记录原始大小
2. 通读全文，标记：[删]常识/重复 [留]个人化/反直觉 [合]重复意图
3. 按类型策略重写
4. `wc -c` 对比，报告删减率
5. 验证：关键选择器/字段ID/端点 是否还在

### 类型C：文档dump混有非直觉知识（如research-paper-writing 67KB）

特征：大部分是领域常识+模板代码，但夹杂少量反直觉/个人化发现
策略：**提炼非直觉部分**，砍85-90%

- "如何写Introduction"等常识 → 删，模型训练过
- LaTeX模板代码块（tikz, algorithm2e） → 删，模型会生成
- 会议要求逐条列举 → 压缩为速查表
- **保留**：反直觉决策表（如autoreason策略选择）、特定公式（如5句abstract公式）、工具集成用法

### 批量处理策略

处理多个大文件时：
- ⚠️ **不要用delegate_task并行处理大文件** — glm-5.1等有速率限制的provider，3个子agent同时读大文件会撞429，全部失败
- 改用**顺序直接处理**：读→写→验证→下一个
- 如果provider无速率限制，delegate_task并行仍可用

## 实战案例

| 文件 | 原始 | 精简后 | 删减率 | 类型 |
|------|------|--------|--------|------|
| pytorch-fsdp | 160KB | 3KB | -98% | A:纯文档dump |
| zsxq-auto | 51KB | 7.9KB | -84% | B:实战积累 |
| research-paper-writing | 67KB | 7KB | -89% | C:dump混非直觉知识 |
| **合计** | **278KB** | **17.9KB** | **-94%** | |

## 踩坑

1. **不要删working code snippet**：代码块是可执行的知识，不是bloat。删代码外的解释，不删代码本身
2. **字段ID/选择器是命脉**：`flduei9oKf` 这种ID删了就要重新查API，代价极高
3. **带日期的踩坑条目有信息量**：`(2026-04-10)` 说明是近期实战验证过的，比无日期的通用规则更可信
4. **同一流程出现两次≠冗余**：可能是不同场景的变体（如"新建表"vs"补全已有表"），先确认意图不同再决定合并还是保留
5. **链接字段格式等细节不能删**：飞书链接字段的两种类型（text+url vs field_type=15）传错格式会报错，这种是反直觉的个人化知识
6. **delegate_task并行大文件处理会撞429**：有速率限制的provider（如glm-5.1），3个子agent同时读大文件必撞限流。改顺序直接处理
7. **references/子目录的文件不动**：SKILL.md中的`[references/xxx.md]`链接指向的文件，精简时只管SKILL.md本身，不删references内容
