## 平台特征
- 综合科技创投媒体，AI 是热门板块但非纯 AI 站
- **JS 动态渲染 SPA**，curl + regex 提取不到有效内容
- **web_fetch / curl 无法提取新闻列表**，必须用 CDP Proxy 的 `/eval` 提取 DOM
- AI 新闻混杂在创投/商业新闻中，需要关键词过滤

## 有效模式
- URL: https://36kr.com/information/AI/ （AI频道）
- CDP `/new` 打开页面后，用 `/eval` 提取：
  ```javascript
  // 获取所有链接，按 AI 关键词过滤
  const aiKeywords = ['AI','人工智能','大模型','GPT','Claude','Gemini','DeepSeek','LLM','智谱','通义','文心','Kimi','豆包','Sora','Agent','机器学习','开源模型','芯片','算力'];
  document.querySelectorAll("a[href]")
    .filter(a => aiKeywords.some(kw => a.textContent.includes(kw)))
  // 取 textContent 和 href，去重
  ```
- 链接格式：`/p/XXXXXXX.html`，需补全为 `https://36kr.com/p/XXXXXXX.html`

## 已知陷阱
- AI 新闻占比中等，需要关键词过滤去掉非 AI 内容
- 页面有大量热门推荐和广告，需按 title 长度过滤（<8字的大概率是导航/标签链接）
- 链接是相对路径，必须补全域名
- 36kr 反爬较强，CDP 访问时偶尔触发验证码，需要重试
- Python3 在 Git Bash 中对中文 URL 编码输出空字符串，URL 编码必须用 Node.js
