## 平台特征
- 综合科技创投媒体，AI 是热门板块但非纯 AI 站
- **JS 动态渲染 SPA，但 SSR 内容已可用**：2026-04-23 验证，curl 直抓可拿到 124KB HTML，Python 正则 `(/p/\d+)` 模式提取到 39 条 AI 相关新闻
- curl 不可用时降级为 [[CDP|CDP]] Proxy 的 `/eval` 提取 DOM
- AI 新闻混杂在创投/商业新闻中，需要关键词过滤

## 有效模式
- URL: https://36kr.com/information/AI/ （AI频道）
- **curl 直接抓取**（2026-04-23 验证可用，首选）：
  ```bash
  curl -s -m 30 -L -H "User-Agent: Mozilla/5.0 ..." \
    "https://36kr.com/information/AI/" -o 36kr.html
  ```
  Python 正则提取：`re.finditer(r'<a[^>]*href=["\x27](/p/\d+)["\x27][^>]*>(.*?)</a>', html, re.DOTALL)`，过滤 title 长度>8 且含 AI 关键词
- **降级方案**：[[CDP|CDP]] `/new` 打开页面后，用 `/eval` 提取：
  ```javascript
  // 获取所有链接，按 AI 关键词过滤
  const aiKeywords = ['AI','人工智能','大模型','GPT','Claude','Gemini','DeepSeek','LLM','智谱','通义','文心','Kimi','豆包','Sora','Agent','机器学习','开源模型','芯片','算力'];
  document.querySelectorAll("a[href]")
    .filter(a => aiKeywords.some(kw => a.textContent.includes(kw)))
  // 取 textContent 和 href，去重
  ```
- 链接格式：`/p/XXXXXXXXX`（无 `.html` 后缀），需补全为 `https://36kr.com/p/XXXXXXXXX`

## 已知陷阱
- AI 新闻占比中等，需要关键词过滤去掉非 AI 内容
- 页面有大量热门推荐和广告，需按 title 长度过滤（<8字的大概率是导航/标签链接）
- 链接是相对路径，必须补全域名
- 36kr 反爬较强，[[CDP|CDP]] 访问时偶尔触发验证码，需要重试
- Python3 在 Git Bash 中对中文 URL 编码输出空字符串，URL 编码必须用 Node.js
