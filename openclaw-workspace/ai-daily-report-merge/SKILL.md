# AI热点日报融合技能

> 前置依赖：`ai-daily-report` skill。分级体系（S/A/B/C + 三筛 + 偏好权重）和风格标准（标题/正文/产品/知识库）均定义在该 skill 中，本 skill 只写 **merge 特有逻辑**。

## ⛔ 强制规则（不可跳过）

**每条新闻必须先分级再写入。** 分级定义和风格标准见 ai-daily-report skill。额外约束：

- S级上限 **2条**，必须是「今天花1小时配置，以后每天省1小时」的杠杆项
- S级核心素材：免费API/开源模型 + 有实操指导的方法论
- 特定场景产品/工具推荐归B/C级，不占S/A（普适性洞见 > 情境性观点）
- 分级宁可偏紧不要偏松——B/C级被错升到A/S比反过来危害更大
- 思维模式 > 具体结论，可迁移智慧 > 专属经验
- 写完全文后必须执行 STOP GATE 自检（见文末）

## 输入

| 来源 | 位置 | 说明 |
|------|------|------|
| 自动日报 | `03-内容工厂/日报/AI热点日报/YYYY-MM-DD-AI热点日报.md` | ai-daily-report 产出 |
| 手工补充 | `D:\cybertomato\03-内容工厂\日报\补充信息\` | 文件名 `YYYYMMDDHHMMSS--来源名.md`，一天可有多个文件，搜索用 glob `YYYYMMDD*` |

补充信息目录无当天文件时，主动询问用户提供。

## 流程

### Step 1：读取 + 去重比对

- 先检查自动日报**内部重复**（不同板块可能报道同一事件）
- 再与补充信息逐条比对，标记：重复项、新增项、差异项

### Step 2：合并

- 同一事件 → 合并为一条，取信息量更高的描述，补充差异细节
- 同一公司/主题多条 → 合并为专题
- 补充信息独有内容 → 按重要性分级入库

### Step 2.5：自动日报板块 → 融合日报分级映射

| 自动日报板块 | 通常映射 | 升级条件 |
|-------------|---------|---------|
| 🔥 今日重点 | ⚡A级 | — |
| 🛠️ 产品与技术 | 📰B级 | 今天就能用的工具→S级；新模型首发+评测夺冠→A级 |
| 💬 业界观点 | 🧠C级 | 有实操指导→A级 |
| 🇨🇳 国内动态 | 📰B级 | 可直接使用的国产工具→S级；新模型发布→A级 |
| 📌 值得关注 | 🧠C级 | 教程/部署指南今天可用→S级 |

判断标准：以「个人实践者今天能用吗」为S级门槛，「能提升工作能力吗」为A级门槛。行业大事归A/B，不占S级。

### Step 3：分级 + 写入

按 ai-daily-report 的分级体系和风格标准写入融合版。

输出路径：`03-内容工厂/日报/Ai融合日报/YYYY-MM-DD-番茄AI热点日报.md`

**融合日报条目标题不编号**（S-1、A-1等），只保留 emoji 分级标记（🎯⚡📰🧠）。编号对读者是噪音。

品牌签名（文末可选）：`*番茄AI热点日报 · 每日精选，不废话*`

## 常见补充信息源

| 来源 | 处理方式 |
|------|---------|
| 观猹每日上新 | 补功能描述，归B级。能访问链接查真实描述；否则基于名称推断并标注 |
| 通往AGI知识库 | 补价值点，归C级 |
| AI热度飙升榜 | 与日报去重合并，新增项按重要性分级 |
| 语鲸日报 | 可解析时交叉参考；只有短链+开场白时直接跳过 |

## 不做的事

- ❌ 正文不放融合说明表/融合时间等元信息
- ❌ 不编造新闻——只有名字没描述的产品，基于名称合理推断并标注
- ❌ 不为凑数降级——新闻少时每板块可缩减到2条

## Step 5（可选）：更新飞书多维表格信息源

`lark-cli base +record-upsert` 写入新增源。字段映射：

| 字段 | 值 |
|------|---|
| 名称 | text（来源名） |
| 类型 | text（聚合源/原创/社区/X账号/搜索关键词/知识库） |
| 备注 | text（一句话价值描述） |
| 链接 | url |
| 层级 | text（Tier1/Tier2） |
| 添加日期 | datetime |

只写新增源，不重复写入。知识库类归Tier1，聚合源归Tier2。

## Step 6（可选）：发布到知识星球

用 `zsxq-publish` skill。同一篇发多个星球，每个独立走一遍。Quill编辑器星球需先切Markdown模式。

## STOP GATE（写完后必须逐条检查）

1. ⛔ 每条都有 🎯/⚡/📰/🧠 标签？
2. ⛔ S级 ≤ 2条？且真的是「今天就能用」？
3. ⛔ A级 3-6条？没有凑数升级？
4. 标题是金句式并列句？
5. 每条有 2-3句摘要 + 1句「为什么重要」？
6. S级 3-5句说透？
7. 产品有条功能定位？知识库有条价值点？
8. 正文无融合元信息？

**任一项不通过→回去修正，不能输出。**
