# Markdown呼吸感排版

将密集文本转为短行留白风格

## 排版风格定义

### 核心原则

**每句独占一行**

逗号处断行

句号处断行

分号处断行

让每个短句有自己的呼吸空间

**段落间空行**

至少一个空行

重要转折处两个空行

视觉上要有"呼吸感"

**行末标点克制**

大多数行末不加标点

句号→去掉

逗号→换成换行

保留冒号（引导下文）

保留破折号（解释说明）

保留问号和感叹号

**禁止事项**

禁止长段落挤在一起

禁止连续三句以上同行

禁止每行超过20个汉字（特殊情况除外）

禁止Markdown加粗/斜体滥用

## 排版规则

### 断行规则

**必须断行的位置**：
- 句号后
- 逗号后（如果前半句已超过10字）
- 破折号前后
- 冒号后（引导列表时）
- "不是...而是..."结构
- "当你...你的..."结构

**可以断行的位置**：
- 连词前（所以、但是、因此）
- 举例说明时
- 数据前后
- 引用前后

**禁止断行的位置**：
- 数字和单位之间
- 人名中间
- 专有名词中间

### 留白规则

**一个空行**：
- 段落之间
- 同一话题的不同角度之间

**两个空行**：
- 话题转换时
- 重要的独立观点前
- 引用或数据之后

### 保留标点的场景

```
冒号——引导下文

问号——提问

感叹号——强调

破折号——解释说明
（前后各一个空格）

省略号——语意未尽
```

### 标点转换表

| 原标点 | 处理方式 | 示例 |
|--------|----------|------|
| 句号。 | 去掉，换行 | 原文：这事很重要。 → 这事很重要\n\n |
| 逗号， | 去掉，换行 | 原文：这事很重要，因为... → 这事很重要\n因为... |
| 分号； | 换行 | 原文：A；B → A\n\nB |
| 顿号、 | 保留或换行 | 并列项少于3个保留，多于3个换行 |
| 引号"" | 保留 | 引用内容不加改动 |
| 括号() | 保留 | 括号内内容保持完整 |

## 执行步骤

### 1. 输入检查

确认输入文本不为空

识别文本语言（中文/英文/混合）

**Success criteria**: 文本已加载

### 2. 分析结构

识别段落边界

识别句子边界

标记需要保留标点的位置

标记需要加空行的位置

**Success criteria**: 结构分析完成

### 3. 执行排版

按断行规则拆分句子

按留白规则插入空行

按标点转换表处理标点

**Success criteria**: 排版后的文本无连续三句同行

### 4. 输出

直接输出排版后的文本

不保留原文对照（除非用户要求）

**Success criteria**: 用户满意排版效果

## 适用场景

- 公众号文章排版
- 飞书文档排版
- 社交媒体文案排版
- 任何需要"呼吸感"的中文长文本

## 不适用场景

- 技术文档（保持标准格式）
- 代码注释（保持紧凑）
- 英文学术论文（保持标准格式）
- 表格数据（保持结构）
