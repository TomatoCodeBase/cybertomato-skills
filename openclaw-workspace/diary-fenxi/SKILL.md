# 日记解析分析器

## 触发条件

| 用法 | 说明 |
|------|------|
| 日记分析 / 日记统计 | 分析今天 |
| 日记分析 2026-04-12 | 分析指定日期 |
| 日记分析 本周 | 分析本周 |
| 日记分析 本月 | 分析本月 |

## 前置条件

- 日记文件存在，格式为Markdown
- 日记文件名格式：`YYYY-MM-DD.md`

## 任务格式识别

### 三种任务形态

| 形态 | 示例 | 解析策略 |
|------|------|----------|
| **定时任务** | `- [ ] 09:15丨任务名丨2026-04-12 10:34:17丨2026-04-12 10:40:23丨Agent名` | 解析耗时、Agent、并行检测 |
| **Agent自动完成** | `- [x] 06:15 丨 AI热点日报采集` | 只计完成率，不计时 |
| **简单任务** | `- [ ] 07:00 丨 吃早餐` / `- [x] 07:00 丨 吃早餐` | 只计完成率，不计时 |

### 行格式正则

```
^- \[(x| )\] (\d{2}:\d{2})\s*丨\s*(.+?)(?:\s*丨\s*(.+?))?(?:\s*丨\s*(.+?))?(?:\s*丨\s*(.+?))?$
```

捕获组：1.完成状态 2.计划时间 3.任务名(可能含双链) 4.开始时间 5.结束时间 6.Agent名

### 时间格式

| 格式 | 示例 | 正则 |
|------|------|------|
| 输入法格式 | `2026年4月12日10:34:17` | `\d{4}年\d{1,2}月\d{1,2}日\d{2}:\d{2}:\d{2}` |
| 标准格式 | `2026-04-12 10:40:23` | `\d{4}-\d{2}-\d{2}\s+\d{2}:\d{2}:\d{2}` |

### 任务名称清洗

```
[[#赛博番茄|公众号「赛博番茄」选题/写作]] → 公众号「赛博番茄」选题/写作
[[#头条打卡]] → 头条打卡
```

## 分析维度

### 1. 每日摘要

完成率、定时任务数、Agent执行数、并行任务数、并行率

### 2. 任务耗时分析

- 任务耗时 = end - start
- 串行总时间 = Σ(任务耗时)
- 并行墙钟时间 = 合并重叠区间后的实际时间
- 并行率 = 串行总时间 ÷ 墙钟时间

### 3. 并行检测算法

```
1. 按开始时间排序所有有时间的任务
2. 遍历：当前.start < 上一个.end → 并行
3. 合并重叠区间计算墙钟时间
```

### 4. Agent效能统计

按Agent分：任务数、平均耗时、并行贡献

### 5. 周/月趋势

每日完成率折线、平均并行率、Agent执行占比、高频跳过任务

## 输出

分析完成后写入文件，同时在对话中输出摘要。

| 类型 | 文件名格式 |
|------|-----------|
| 日报 | `{YYYY-MM-DD}-日记分析.md` |
| 周报 | `{YYYY}-W{ww}-周报.md` |
| 月报 | `{YYYY}-{MM}-月报.md` |

## 并行执行认知

多Agent并行执行是常态：
1. 墙上时间 ≠ 实际占用时间
2. 不可简单加总时间
3. 分析时默认假设存在并行，标注"并行率"
4. 区分"专注时间"和"挂机时间"
5. 完成率高 + 并行率高 = 高效

## 边界情况

- 文件不存在 → 提示无日记
- 时间格式混合 → 统一转换
- 无时间已勾选 → 只计完成
- 任务名含双链 → 提取显示文本
- 临时事项格式更自由 → 尽力解析
