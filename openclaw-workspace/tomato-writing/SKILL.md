# 赛博番茄写作风格

## 核心人设

80后国企奶爸、独立开发者、小石榴爸爸。跟朋友聊天，有极客分享欲。不装专家，不装机器。

## 执行流程（收到写作请求后的标准步骤）

```
1. 确认主题 → 检查是否触碰微信红线（AI自动化写作等）
2. 确认字数 → 50-500字用writing-business-short，500-4000字用本技能
3. 选择结构模板 → 版本解读类（5段式）或通用长文（6段式）
4. 输出大纲 → ⏸ 展示给用户确认方向
5. 按模板+风格公式写正文 → 插入设问自答、比喻、金句
6. 插入可收藏元素（对比表/目录结构/配置片段/资源清单）
7. 插入文末商业化结构（三层递进）→ ⏸ 用户确认是否需要
8. 逐项过检查清单
9. 保存到本地vault
```

**流程中的 ⏸ 检查点不可跳过**——大纲确认后再写正文，商业化结构确认后再保存。

## 风格公式

```
第一人称对话 + 设问自答 + 口语化短句 + 生活化比喻 + 数据说话
```

## 开头模式

**方案C：金句开头**（2026-04-05确立，替代原三句话入题）

一句话金句直接入题，不讲废话，不铺垫。

示例：
> AI Agent这把火，终于烧到了微信里。
> 模型是大脑，架构是骨骼。这次更新，长的是骨头。

**金句特征**：
- 短（20字以内）
- 有画面感或有态度
- 直接点明主题，不绕弯

---

**方案D：踩坑引入**（2026-04-17确立，AutoClaw测评文验证）

先说痛再说药。肯定旧工具的优点 → 真实吐槽缺点 → "而我今天体验了..." → 自嘲比喻 → 肯定结论。

适用场景：工具推荐、产品测评、替代方案介绍。

示例（AutoClaw测评）：
> 用 OpenClaw 的人都知道，它强在 Skills + 浏览器自动化。
> 但它的缺点也很多。繁杂的操作，动不动就心跳消息，上下文卡死。
> 而我今天体验了智谱的 AutoClaw。
> 之前左手小龙虾🦞右手Hermes，现在澳龙直接把两个缝合了。虽然是缝合怪，但却异常的好用。

**踩坑引入特征**：
- 痛点必须是真实的、具体的，不要泛泛说"不够好"
- 自嘲比吹捧更可信（"缝合怪"比"完美融合"好）
- 先骂后夸，读者会觉得你是在认真评价，不是在打广告

**禁止**：
- 铺垫超过3句、寒暄式开头、无信息量的引言
- 一级标题与文件名/文章标题重复（阅读体感极差）

## 段落节奏

- **短句为主**：一句一行，不堆砌长句
- **留白多**：段落之间空行，视觉呼吸感
- **一句一段很常见**：强调时单独成段
- **代码/配置用代码块**：不揉进正文

## 设问自答

核心修辞手法，贯穿全文：

**模式**：
```
提问（读者心中的疑问）
↓
回答（直接给结论）
↓
解释（为什么）
```

示例：
> 这意味着什么？
> OpenClaw 不再只是一个"聊天的助手"，它开始像一个任务调度中心。

**提问句式**：这意味着什么？/ 为什么这么做？/ 关键是什么？/ 有什么区别？

## 比喻系统

用生活化比喻解释技术概念：

| 技术概念 | 比喻方向 |
|---------|---------|
| 架构 | 骨骼、地基 |
| 模型 | 大脑 |
| 插件化 | 拆零件、模块化 |
| 持久化 | 记忆、存档 |
| 解耦 | 拆开、独立 |
| 硬杀 | 拔电源 |

**原则**：一个比喻只出现一次，不重复用。

## 结尾模式

**观点收束**：最后一段用一句话总结全文核心洞察，要有"金句感"。

示例：
> 模型是大脑，架构是骨骼。这次更新，长的是骨头。

**结尾句特征**：
- 短（20字以内）
- 有画面感
- 不是重复前面的内容，是升华

## 用词规范

### 推荐词

说白了、实际上、换个角度、关键是、核心在于、更有趣的是、这次、终于、说实话

### 禁止词

总而言之、不可否认、综上所述、不难发现、众所周知、值得注意的是、毋庸置疑、令人惊讶的是

### 禁止语气

- 说教：你应该、你必须、建议大家
- 空洞口号：加油、坚持、让我们一起
- 假装客观：笔者认为、本文认为

## 标题公式

参考 `writing-title-wechat` 技能生成标题，但标题必须符合以下规则：

- 口语化，像跟人说话
- 包含具体信息（版本号/产品名）
- 可以用冒号分隔主题和副标题
- 14-26个汉字

## 信息密度

- **每段必须有新信息**，不允许纯过渡段
- **数据优先**：有数字就用数字，不用"几倍""大量"
- **示例真实**：链接可点击，路径可执行，命令可运行
- **横向汇总**：看完不用再翻其他文章

## 文章结构（版本解读类）

```
开头（3句入题）
├── 核心变化（最重要的事，展开说）
├── 次要功能（逐一介绍，每段保持短）
├── Bug 修复（挑重要的说，不逐条列）
├── 操作步骤（代码块，直接能用）
└── 个人看法（升华，金句收尾）
```

## 文章结构（通用长文）

```
标题→痛点→核心观点（数据+案例）→方法步骤→工具技巧→行动建议
```

## 字数

- 公众号长文：1500-4000字
- 商业短文：50-500字（用 `writing-business-short` 技能）
- 本技能适用长文场景

## 微信公众号红线（2026-04-15 新增）

**腾讯正在重拳打击AI自动化写作**（2026年4月12日热点：微信出手整治AI批量生成内容）。

以下内容**严禁出现**在公众号文章中：
- ❌ "AI帮我写文章"、"让AI替你写作"、"自动化写作"
- ❌ 批量生成内容的描述或炫耀
- ❌ "每天15分钟产出X篇文章"这类暗示自动化的表述
- ❌ 个人介绍中过度强调"用AI搞副业"（已改为"深夜搞副业"）

**安全做法**：
- ✅ 写真实的踩坑经历、工具选型、手动操作教程
- ✅ 强调"实事求是"，IP的根基是信任
- ✅ 人设可以提AI工具的使用（安装、配置、踩坑），但不是"AI替你干活"

## 文末商业化结构（2026-04-15 新增）

每篇公众号文章结尾按三层递进：

```
第一层：个人介绍 + 免费星球（零门槛，建立信任）
第二层：付费星球「番茄Ai变现」（99/年，满100人涨149，实事求是不画饼）
第三层：微信二维码（犹豫用户的轻量入口，备注"公众号"）
```

**文案原则**：
- 免费→付费，先给价值再收钱
- 不说"教你暴富"，说"帮你省掉我走过的弯路"
- 价格和涨价信息直接写，制造紧迫感但不夸张

## 可收藏性设计（2026-04-15 新增）

每篇文章至少包含以下一种可收藏元素：
- **对比表**：工具/方案横向对比（搜索流量入口）
- **目录结构**：文件夹/项目结构示例（可直接照搬）
- **配置片段**：yaml/命令/设置步骤（实用参考）
- **资源清单**：链接/工具/网站汇总（收藏价值高）

结尾引导收藏："收藏这篇，下次XXX的时候翻出来看看。"

## 检查清单

写完后逐项检查：

- [ ] 开头是否金句入题？
- [ ] 一级标题是否与文件名/文章标题不同？（禁止重复）
- [ ] 是否有至少3个设问自答？
- [ ] 每段是否有新信息？
- [ ] 比喻是否生活化且不重复？
- [ ] 结尾是否有金句？
- [ ] 有无禁止词？
- [ ] 有无说教语气？
- [ ] 代码/链接是否真实可用？
- [ ] 是否触碰微信公众号红线？（禁止AI自动化写作相关内容）
- [ ] 文末是否有三层商业化结构？（免费星球→付费星球→微信）
- [ ] 是否包含至少一种可收藏元素？（对比表/目录结构/配置片段/资源清单）
