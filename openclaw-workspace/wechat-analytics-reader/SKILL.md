# 微信公众号数据分析读取器

## 背景

微信公众号后台导出的趋势数据文件（如 `tendency_xxx_xxx.xls`）包含三个区域：
1. 趋势概况 — 每日各渠道阅读人数
2. 趋势分析 — 每日新增阅读/分享/收藏/发文章数
3. 数据来源概况 — 按文章+渠道维度的阅读明细

## 关键问题：编码

Windows 上 xlrd 读取这些 xls 文件时，终端（Git Bash/MSYS2）无法正确显示中文字符，即使 pandas/xlrd 内部解码正确（Unicode 码点正确，终端渲染失败）。

### 解决方案：写 UTF-8 文件再读

```python
/d/RJCX/Python/python.exe -c "
import pandas as pd
df = pd.read_excel('D:/path/to/tendency_xxx.xls')

with open('D:/path/to/temp_output.txt', 'w', encoding='utf-8') as f:
    # Section 1: 趋势概况 — cols 1-3 (日期, 渠道, 阅读人数), 行2开始
    # Section 2: 趋势分析 — cols 5-9 (日期, 分享人数, 跳转阅读原文, 微信收藏, 发表篇数), 行2开始
    # Section 3: 阅读来源分析 — cols 11-15 (传播渠道, 发表日期, 内容标题, 阅读人数, 阅读人数占比), 行2开始
    # source=='全部' 的行是文章总阅读，其余是分渠道明细
    ...
"
```

然后用 `read_file` 读 temp_output.txt。

## Python 路径

本机 hermes venv 没有 pandas，系统 Python `/d/RJCX/Python/python.exe` 有 pandas 3.0.1。

**始终用 `/d/RJCX/Python/python.exe` 执行 pandas 脚本**，不要用 venv 的 python。

## 数据布局

| 区域 | 列索引 | Row 1 表头 |
|------|--------|-----------|
| 趋势概况标题 | col 1 | 数据趋势概况(日期范围) |
| 趋势概况 | col 1=日期, 2=渠道, 3=阅读人数 | 日期, 渠道, 阅读人数 |
| 趋势分析标题 | col 5 | 数据趋势概况(日期范围) |
| 趋势分析 | col 5=日期, 6=分享人数, 7=跳转阅读原文, 8=微信收藏, 9=发表篇数 | 日期, 分享人数, 跳转阅读原文人数, 微信收藏人数, 发表篇数 |
| 来源分析标题 | col 11 | 数据来源概况(日期范围) |
| 来源分析 | col 11=传播渠道, 12=发表日期, 13=内容标题, 14=阅读人数, 15=阅读人数占比 | 传播渠道, 发表日期, 内容标题, 阅读人数, 阅读人数占比 |

Row 0 = 空行, Row 1 = 区域标题行, Row 2 = 列标题行, Row 3+ = 数据行。

## 渠道类型

趋势概况中的渠道：公众号消息、聊天会话、朋友圈、公众号主页、其他、推荐、搜一搜、全部

来源分析中的渠道：全部、朋友圈、搜一搜、聊天会话、公众号消息、公众号主页、推荐、会话、更多

**注意**：来源分析中 `source=='全部'` 的行是文章级别的总阅读，非渠道拆分。做文章排行时只取 `source=='全部'`。

## 常用分析

### 每日总阅读趋势
过滤 趋势概况 中 channel=='全部' 的行。

### 文章排行
取 来源分析 中 source=='全部' 的行，按 reads 降序，去重（同文章同日期可能出现在不同区域）。

### 关键指标
- 日均阅读 = 全部阅读人数总和 / 天数
- 发文频率 = 发表篇数总和 / 天数
- 收藏率 = 微信收藏总和 / 阅读总和
- 搜一搜占比 = 搜一搜来源阅读 / 总阅读
